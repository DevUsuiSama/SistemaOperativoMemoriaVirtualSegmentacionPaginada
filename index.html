1. Memoria Virtual

La memoria virtual es una técnica que permite ejecutar procesos cuyo tamaño excede la memoria física disponible, utilizando almacenamiento secundario para guardar las páginas que no caben en memoria RAM.

Explicación del Proceso:

La memoria virtual permite que los programas utilicen más memoria de la físicamente disponible. Cuando se accede a una página que no está en memoria física, ocurre un page fault y el sistema debe traer la página desde el disco, reemplazando otra si es necesario.

2. Tablas de Páginas

La tabla de páginas es la estructura que mapea direcciones virtuales a físicas. Cada proceso tiene su propia tabla de páginas gestionada por el sistema operativo.

Proceso de Traducción:

1. La dirección virtual se divide en número de página y desplazamiento
2. Se consulta la tabla de páginas para obtener el marco físico
3. Si la página no está en memoria (bit de presencia=0), ocurre un page fault
4. El marco físico se combina con el desplazamiento para obtener la dirección física

3. Segmentación vs. Paginación

Dos enfoques diferentes para la gestión de memoria: paginación (tamaño fijo) vs segmentación (tamaño variable).

Diferencias Clave:

Segmentación: Unidades lógicas de tamaño variable, refleja la estructura del programa, sufre de fragmentación externa.
Paginación: Unidades físicas de tamaño fijo, transparente al programador, sufre de fragmentación interna.

Ventajas del Modelo Híbrido:

• Protección: Cada segmento tiene permisos específicos
• Eficiencia: No hay fragmentación externa
• Organización lógica: Los programas se dividen en segmentos con significado
• Compartición: Es fácil compartir segmentos entre procesos
